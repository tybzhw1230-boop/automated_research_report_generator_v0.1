# 项目信息传递链路全面分析

本文档只记录当前已经落地的 Flow → Crew → Registry → Markdown/PDF 链路，不再展示已经退场的 planning crew、question tree 或 research scope 设计。

---

## 一、当前 Flow 执行拓扑

```text
@start() prepare_evidence
    │  PDF解析 → 元数据落盘 → 页索引生成 → 当前 run 目录初始化 → registry 初始化
    │  输出: company_name / industry / page_index / registry_path
    ▼
@listen(prepare_evidence) build_research_plan
    │  加载 flow/config/registry_template.yaml
    │  替换 {company_name} / {industry}
    │  重建当前 run 的 evidence_registry.json
    ▼
@router(build_research_plan) run_research_crew
    │  顺序执行 7 个 research sub-crews
    │  每个 crew: search_facts → extract_file_facts → check_registry → synthesize_and_output
    │  输出: 7 个 research packs + registry 更新
    ▼
@listen(RESEARCH_STAGE_COMPLETED) review_research_gate
    │  QACrew 做 research 外部 QA
    │  输出: GateReviewOutput
    │  Flow 调 apply_gate_review() 回写 registry
    ▼
@router(review_research_gate) route_research_gate
    ├─ pass / force_pass ───────────────────────────┐
    └─ retry → rerun_research(仅重跑 affected_packs) │
               → 回到 review_research_gate          │
                                                    ▼
@router(or_("research_gate_passed", RESEARCH_GATE_FORCE_PASS_EVENT)) run_valuation_crew
    │  ValuationCrew 输出 3 个 valuation packs
    ▼
@router(VALUATION_STAGE_COMPLETED_NO_GATE_EVENT) run_investment_thesis_crew
    │  InvestmentThesisCrew 输出 thesis + diligence questions
    ▼
@listen(THESIS_STAGE_COMPLETED_NO_GATE_EVENT) publish_if_passed
    │  WriteupCrew 汇编最终 Markdown 并导出 PDF
    ▼
    END
```

---

## 二、阶段输入输出总览

### 1. `prepare_evidence`

输入：

- `pdf_file_path`

主要动作：

- 调用文档元数据识别
- 为当前公司生成 `run_slug`
- 创建 `.cache/<run_slug>/indexing/`、`.cache/<run_slug>/logs/`、`.cache/<run_slug>/md/`
- 在当前 run 下保存 document metadata 和 page index
- 初始化 registry，并把上下文注入工具层

关键输出：

- `document_metadata_file_path`
- `page_index_file_path`
- `evidence_registry_path`
- `run_debug_manifest_path`

### 2. `build_research_plan`

输入：

- `company_name`
- `industry`
- `flow/config/registry_template.yaml`

主要动作：

- 加载固定 YAML 模板
- 替换模板中的 `{company_name}`、`{industry}`
- 用模板条目覆盖当前 run 的 registry
- 写 checkpoint `cp01_planned`

关键输出：

- research registry 已就绪
- owner_crew 分布已可追踪

### 3. `run_research_crew`

统一公共输入来自 `_base_inputs()`：

- `company_name`
- `industry`
- `pdf_file_path`
- `page_index_file_path`
- `document_metadata_file_path`
- `document_profile_summary`

每个 sub-crew 额外还会收到：

- `pack_name`
- `pack_title`
- `owner_crew`
- `pack_output_path`
- `loop_reason`
- `qa_feedback`
- 各 crew 的 `search_guidance`、`extract_guidance`、`qa_guidance`、`synthesize_guidance`

当前执行顺序与输出：

1. `history_background_crew` → `01_history_background_pack.md`
2. `industry_crew` → `02_industry_pack.md`
3. `business_crew` → `03_business_pack.md`
4. `peer_info_crew` → `04_peer_info_pack.md`
   - 额外依赖 `industry_pack_text`、`business_pack_text`
5. `financial_crew` → `05_finance_pack.md`
   - 额外依赖 `peer_info_pack_text`
6. `operating_metrics_crew` → `06_operating_metrics_pack.md`
   - 额外依赖 `peer_info_pack_text`
7. `risk_crew` → `07_risk_pack.md`

Flow 后处理：

- 每个 pack 完成后都会调用 `_register_pack_output()`
- pack 正文前 800 字会作为 `context evidence` 登记到对应 judgment entry
- 每个 pack 都会写单独 checkpoint

### 4. `review_research_gate`

输入：

- 7 个 research packs 拼成的 `stage_bundle`
- `registry_markdown_text`
- `registry_diff_text`
- `registry_full_text`

主要动作：

- 运行 `QACrew`
- 当前只做 research 阶段外部 QA
- consistency 检查当前延后为占位结果，主 gate 以 coverage 结果驱动

当前路由规则：

- `pass` → 直接进入 valuation
- `revise` → 只重跑 `affected_packs`
- 超过上限 → `force pass`

### 5. `run_valuation_crew`

输入：

- `peer_info_pack_text`
- `finance_pack_text`
- `operating_metrics_pack_text`
- `risk_pack_text`

输出：

- `01_peers_pack.md`
- `02_intrinsic_value_pack.md`
- `03_valuation_pack.md`

说明：

- `peers_pack` 和 `intrinsic_value_pack` 会登记到 registry
- 当前 valuation 阶段不再经过外部 gate

### 6. `run_investment_thesis_crew`

输入：

- 全部 7 个 research packs
- `peers_pack_text`
- `valuation_pack_text`
- `registry_full_text`

输出：

- `01_investment_thesis.md`
- `02_diligence_questions.md`

说明：

- thesis 当前可以读取完整 registry
- 当前 thesis 阶段不再经过外部 gate

### 7. `publish_if_passed`

输入：

- 全部 research packs
- `valuation_pack_text`
- `investment_thesis_text`
- `diligence_questions_text`
- `final_qa_summary`
- `final_report_markdown_path`
- `final_report_pdf_path`

输出：

- 最终 Markdown
- 最终 PDF
- `cp06_writeup` checkpoint

说明：

- writeup prompt 当前要求对上游正文做“完整插入”，而不是再次摘要
- 最终 QA 摘要当前只汇总 research 阶段 QA 结果

## 三、Registry 信息流

当前 registry 主文件：

- `.cache/<run_slug>/md/registry/evidence_registry.json`

当前 registry Markdown 快照：

- `.cache/<run_slug>/md/registry/registry_snapshot.md`

当前 registry 模板入口：

- `src/automated_research_report_generator/flow/config/registry_template.yaml`

当前 entry 契约：

- `entry_type`: `fact` / `data` / `judgment`
- `content_type`: `single` / `table`
- `status`: `unchecked` / `checked` / `need_revision`
- `topic`: `history`、`industry`、`business`、`peer_info`、`financial`、`operating_metrics`、`risk`、`peers`、`intrinsic_value`、`valuation`、`investment_thesis`
- `owner_crew`: 当前真实 crew 名

当前稳定工具接口：

- `add_entry`
- `update_entry`
- `add_evidence`
- `status_update`
- `registry_review`
- `read_registry`

当前读取方式：

- `view="markdown"`：分组 Markdown 视图
- `view="entry_list"`：轻量条目列表
- `view="full"`：完整 JSON 快照
- `view="entry_detail"`：按 entry ID 下钻
- `view="evidence_detail"`：按 evidence ID 下钻

当前典型流转：

1. `prepare_evidence` 初始化 registry
2. `build_research_plan` 用模板重建 registry
3. research crews 读取自己负责的 entries，补 `content/source/confidence/status`
4. research crews 追加 evidence，并留下 `registry_review`
5. Flow 把 pack 产物登记为 context evidence
6. `review_research_gate` 根据 `affected_packs` 回写 `need_revision` 或 `checked`
7. thesis 读取 registry 做投资逻辑与尽调问题整理

## 四、运行目录与阶段产物

当前单次 run 的目录结构：

```text
.cache/<run_slug>/
├─ indexing/
├─ logs/
└─ md/
   ├─ research/iter_XX/
   ├─ valuation/iter_XX/
   ├─ thesis/iter_XX/
   ├─ qa/research/iter_XX/
   ├─ registry/
   ├─ checkpoints/
   ├─ run_manifest.json
   ├─ <pdf_stem>_v2_report.md
   └─ <pdf_stem>_v2_report.pdf
```

关键点：

- 文档元数据和页索引不在公共目录里，而是在当前 run 的 `indexing/`
- crew 日志和 flow 日志都在 `logs/`
- research、valuation、thesis、QA 都按 `iter_XX` 分目录
- 当前不再使用项目级 `latest_run.json`

## 五、当前最终报告链路

当前最终报告由 `writeup_crew` 生成，固定一级标题为：

1. 投资逻辑
2. 尽调问题
3. 公司历史及股东
4. 行业分析
5. 业务分析
6. 财务分析
7. 风险分析
8. 可比公司情况
9. 综合估值
10. QA摘要与结论边界

当前写入策略：

- thesis、尽调问题、7 个 research packs、`valuation_pack` 和 `final_qa_summary` 会进入最终报告
- `MarkdownToPdfTool` 负责把最终 Markdown 导出为 PDF
